mobile apps

what types of mobile apps do we know?

NATIVE mobile apps


  • A native app is an app for a certain mobile device (smartphone, tablet, etc.)
  • It is installed directly onto the device
  • Coded in a specific programming language


  • iOS = Objective-c
  • Android = Java (Dalvik VM)
  • Windows Phone = C/C++

NATIVE mobile apps


Advantage:

  • great performance
  • ability to use device-specific hardware
  • no internet needed

Disadvantage:

  • specific platform
  • native languages
  • expensive

NATIVE mobile apps

web mobile apps


  • Internet-enabled apps that have specific functionality for mobile devices.
  • They're accessed through the mobile device's web browser
  • Don't need to be downloaded and installed on the device
  • Use standard web technologies: typically HTML5, JavaScript and CSS

web mobile apps


Advantage:

  • various platforms
  • easier to develope
  • jQuery Mobile & Sencha Touch

Disadvantage:

  • slow performance
  • internet connection
  • cant use device-specific hardware

web mobile apps

combine it?

Advantages:

  • various platforms
  • HTML, Javascript, CSS
  • ability to use device-specific hardware
  • no internet needed

hybrid mobile apps

Hybrid development combines the best (or worst) of both the native and HTML5 worlds. 


  • "native" (can use device-specific hardware)
  • installed in device
  • write in HTML5, CSS, JS
  • cross-platform

hybrid mobile apps

how hybrid works

  • Run inside a native container
  • Use the device's browser engine to render the HTML and process the JavaScript locally
  • A web-to-native abstraction layer enables access to device capabilities (accelerometer, camera, local storage, etc)
